# 🦋 標本レンダリング設定仕様書

本ドキュメントでは、AR空間およびフォトモードにおける標本（3Dモデル）の表示・アニメーション設定について解説します。

---

## 1. 設定の管理構造

標本の設定は、`backend/lib/specimens/` ディレクトリ内で標本ごとに独立したファイルとして管理されています。

- **`types.ts`**: 設定項目のインターフェース定義
- **`index.ts`**: 全標本設定の集約と提供
- **`[標本名].ts`**: 個別の 3D パラメータ定義

---

## 2. 設定パラメータの定義

`SpecimenSettings` インターフェースで定義されている主要な項目は以下の通りです。

| パラメータ                      | 役割               | 説明                                                      |
| :------------------------------ | :----------------- | :-------------------------------------------------------- |
| `scale`                         | 標準スケール       | ARマーカー上で表示される際の基準サイズ                    |
| `outerAnimation`                | 外側アニメーション | モデル全体に対する動き（主に Y軸回転）                    |
| `innerAnimation`                | 内側アニメーション | 浮遊感や伸縮など、モデル固有の微細な動き                  |
| `minScale` / `maxScale`         | オートスケール範囲 | カメラとの距離に応じた自動リサイズの限界値                |
| `releaseScale`                  | リリース用スケール | フォトモード（RELEASE）専用のサイズ（未指定時は `scale`） |
| `release[Outer/Inner]Animation` | リリース用動き     | フォトモード専用のアニメーション（未指定時は標準設定）    |

---

## 3. シーン別の出し分けロジック

アプリケーションの画面に応じて、適用される設定が自動的に切り替わります。

### 🔍 ARスキャン画面 (`/ar`)

- **目的**: 観察と解析のしやすさを優先。
- **挙動**: 標準の `scale` を使用。マーカーの真上で一定のサイズに収まるように調整されます。

### 📸 フォトモード画面 (`/release`)

- **目的**: 現実世界への配置と存在感を優先。
- **挙動**: `releaseScale` 等の上書き設定を優先して使用します。一般的にスキャン時よりも大きく、よりダイナミックな動きに設定されています。

---

## 4. 標本別設定のポイント

現在の主要な標本設定の設計方針です。

- **Leviathan (クジラ)**: リリース時はスキャンの 6倍のサイズに拡大し、悠然とした回遊を表現。
- **Common Blue (蝶)**: リリース時は 2倍のサイズにし、より大きく上下に羽ばたくように設定。
- **Antique Sword (剣)**: 実物大（ライフサイズ）を意識したスケール設定。
- **Great Wave (大波)**: 波のディテールが見えるよう、フォトモードでは大幅に拡大。

---

## 5. 新しい標本の追加手順

1. `backend/lib/specimens/` に新しい `.ts` ファイルを作成する。
2. `types.ts` の `SpecimenSettings` に準拠したオブジェクトを定義する。
3. `index.ts` で作成したファイルをインポートし、`SPECIMEN_SETTINGS` マップに登録する。
